# 重定向

如何实现重定向功能 ?

# 登录重定向

在点击登录页面中中对 $route 进行监听：

watch: {
  $route: {
    handler: function(route) {
      // 获取登录页面路由的 query 参数
      const query = route.query
      if (query) {
        // 保存登陆成功后需要跳转的重定向路由名
        this.redirect = query.redirect
        // 保存登陆成功后需要跳转的重定向路由所携带的参数
        this.otherQuery = this.getOtherQuery(query)
      }
    },
    immediate: true
  }
}

this.getOtherQuery(query) 的用途是获取除 redirect 外的其他查询条件，登录成功后：

this.$store.dispatch('user/login', this.loginForm)
.then(() => {
  this.$router.push({ path: this.redirect || '/', query: this.otherQuery })
  this.loading = false
})
.catch(() => {
  this.loading = false
})

完成重定向的代码为:

this.$router.push({ path: this.redirect || '/', query: this.otherQuery })

上述代码可以保证完成登录后跳转到需要重定向的路由并会携带其所需的参数

# 重定向组件

重定向组件代码如下：

<script>
export default {
  created() {
    const { params, query } = this.$route
    const { path } = params
    this.$router.replace({ path: '/' + path, query })
  },
  render: function(h) {
    return h() // avoid warning message
  }
}
</script>

重定向组件配置了动态路由,实际使用如下：

{
    path: '/redirect',
    component: Layout,
    hidden: true,
    children: [
      {
        path: '/redirect/:path*',
        component: () => import('@/views/redirect/index')
      }
    ]
}

TIP

小细节

path: '/redirect/:path*'

* 表示匹配零个或多个路由，比如路由为 /redirect 时，仍然能匹配到 redirect 组件。如果将路由改为：

path: '/redirect/:path'

此时路由 /redirect 将只能匹配到 Layout 组件，而无法匹配到 redirect 组件
